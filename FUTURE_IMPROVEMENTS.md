# 향후 개선 및 추가 기능 제안

현재 프로젝트의 기능을 분석하여 실용적이고 구현 가능한 개선 사항들을 제안합니다.

## 🎯 우선순위 높음 (즉시 구현 가능)

### 1. 구독 정보 수정 기능
**현재 상태**: 구독 정보를 추가하고 해지만 가능
**개선 내용**:
- 구독 카드에 "수정" 버튼 추가
- 금액, 다음 결제일, 주기 등을 수정할 수 있는 모달/폼
- AI 재분석 기능 (기존 텍스트로 다시 분석)

**구현 난이도**: ⭐⭐ (중간)
**사용자 가치**: ⭐⭐⭐⭐⭐ (매우 높음)

### 2. 결제 내역 히스토리
**현재 상태**: 현재 활성 구독만 표시
**개선 내용**:
- 해지된 구독도 히스토리로 보관
- 월별/연도별 결제 내역 통계
- 그래프로 시각화 (Chart.js 또는 Recharts)

**구현 난이도**: ⭐⭐ (중간)
**사용자 가치**: ⭐⭐⭐⭐ (높음)

### 3. 구독 카테고리/태그 기능
**현재 상태**: 모든 구독이 동일하게 표시
**개선 내용**:
- 카테고리 분류 (스트리밍, 클라우드, 소프트웨어, 뉴스레터 등)
- 태그 시스템 (필수, 선택, 테스트 등)
- 카테고리별 필터링 및 통계

**구현 난이도**: ⭐⭐ (중간)
**사용자 가치**: ⭐⭐⭐⭐ (높음)

### 4. 알림 설정 개선
**현재 상태**: 3일 전 알림만 가능
**개선 내용**:
- 알림 일수 커스터마이징 (1일, 3일, 7일, 14일 등)
- 여러 시점 알림 (예: 7일 전 + 3일 전)
- 알림 채널 선택 (Slack, 이메일, 브라우저 푸시)

**구현 난이도**: ⭐⭐⭐ (중간-높음)
**사용자 가치**: ⭐⭐⭐⭐⭐ (매우 높음)
